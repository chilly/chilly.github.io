因为new String()时在堆中生成了空间给"a"，对于第一次的s=="a"时，栈分配空间给"a"，并把"a"当成一个匿名的对象。赋值ss="a"，意味着在栈中分配了空间给"a"，并把"a"这个匿名的对象引用赋给ss。所以s与"a"的引用不等。而"a"再次出现在代码中时，编译器查找到先前的"a"对象，而不是再分配内存给"a"。所以ss与"a"的引用是相等的。但不管引用是否相等，其值都是"a"，这便看出了equals()的威力。
